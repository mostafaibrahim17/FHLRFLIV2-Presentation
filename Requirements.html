Background

With the development of healthcare industry, people put more and more attention on their heath status. Huge amount of health data is gathered from various approaches, such as gym, wearable equipments, etc, However, it's quite hard for users to make use of it today because it's scattered everywhere. In order to make it easier, we developed FHIR FLI Version 2.0.

Goals

This project aims to unify and gather the health data from several sources and display it on a dashboard. The graphs should be personlized according to users' needs, and the data should be transformed into a unified version. For further consideration, we aim to implement a decentralized data storage with Solid Inrupt.

MoSCoW Requirements List


Must
  • Full fledged working login system (User sign-in & sign-up).
  • Users must be able to see data from the Fitbit in a visualised and interesting way.

Users must be able to signup on the FHIR FLI web application, login using their credentials to a dashboard that presents their health and lifestyle data using graphs, the dashboard also allows user to add certain health goals

Should
  • Personlized dashboard according to the personality of the user

    • The user will be presented with a set of personality questions that will determine what information that user prefers to see and cares about and therefore according to his answers, some information will be filtered out and some will be highlighted

  • More complex and decentralized user data storage system using Solid

    • This can help making storing the users data much more efficient and gives users the ownership of data due to the fact that Solid inrupt offers decentralized databases.

Could
  • Docker-container to make deployment easier for future development

    • Deploying this web application and getting it running to start developing it can be tricky and very time-consuming (which happened with us since we are upgrading it from version 1.0 to 2.0), and so an excellent idea would be to implement a docker container that makes deployment much more simple.
